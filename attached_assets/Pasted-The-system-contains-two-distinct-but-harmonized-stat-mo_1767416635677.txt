The system contains two distinct but harmonized stat models. The first is the Core Attribute system, consisting of STR, AGI, INT, VIT, and SEN. These represent raw capacity and long-term growth and are increased manually through training buttons. Each stat is capped at 500 to enforce realistic progression limits. These attributes are visualized using a radar chart that automatically updates whenever a stat changes, with the chart scale fixed to the stat cap to preserve proportional accuracy.
The second stat model is the Kaizen Hexagon, a six-point radar chart representing weekly or periodic self-evaluation rather than raw accumulation. The six attributes are STR, INT, SPI, VIT, WIS, and DIS, each adjustable via sliders from 1 to 100. Adjusting any slider must instantly reshape the radar chart and trigger a recalculation of overall level. This hexagon represents balance, discipline, and consistency rather than power.
Level calculation must be automatic and derived from the combined total of Core Attributes and a weighted contribution from the Kaizen Hexagon. The level updates instantly whenever any relevant stat changes, ensuring the system never desynchronizes. The Title field must evolve automatically based on stat thresholds, such as progressing from Unawakened to Apprentice Technician, Electrical Engineer, or Systems Architect depending on INT and SEN values. Titles are not manually editable and exist purely as a reflection of progress.
The training module allows users to increase individual Core Attributes by pressing +1 buttons. Each press must validate the stat cap, apply the increment, update the radar chart, recalculate level, and log the action. If a stat has reached its maximum, the system must deny the action and log the reason clearly.
The system includes a physical Daily Quest module that tracks tangible actions such as push-ups, sit-ups, squats, and running distance. Each task has a defined maximum and progresses incrementally through manual input. The reward for this module is only claimable when all required objectives are fully completed. Claiming the reward restores HP and MP to full and grants a flat bonus to all Core Attributes. Attempting to claim the reward prematurely must fail visibly and be logged.
A separate discipline-focused Daily Quest system tracks consistency rather than volume. This system consists of fixed tasks that always appear and a rotating set of randomly selected substitute tasks generated daily. Tasks are completed via checkboxes and tracked internally. Completing at least five tasks allows the user to claim a reward in the form of Fast Track Points (FTP), which represent long-term discipline currency. Claiming the reward resets task completion and regenerates the random tasks.
